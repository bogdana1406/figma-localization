=== KNOWN LIMITATIONS OF THIS PROMPT ===
Based on practical experience with full-deck translation through this per-node API method, two categories of error are NOT reliably fixable through prompt wording alone:

1. Cross-node consistency for identical or near-identical source strings: Because each node is translated independently with no memory of prior nodes, the exact same source sentence appearing twice in the deck can come back with two different Brazilian Portuguese renderings. No prompt instruction fully eliminates this — it is a structural limitation. RECOMMENDED PROCESS: before submitting, deduplicate identical source strings (script1_dedup.py), translate each unique string once, and propagate programmatically back to all occurrences (script2_reexpand.py) — rather than resubmitting duplicates independently. This file has already been deduplicated.

2. Line break collapse: The most common per-node error is collapsing a multi-line source into a single line. Always count \n characters before translating and match the count in the output.

3. Percentage/number formatting: The model understands the rules (Rule 3) but may occasionally output "0.5%" instead of "0,5%", or keep "10,000" instead of converting to "10.000". Run a mechanical post-processing pass after translation to catch these systematically.

=== GENDER DEFAULTS FOR AMBIGUOUS TERMS ===
English source words like "partner", "affiliate", "referral", "user", "member" are gender-neutral. When the source provides no gender context, always use the masculine form as the grammatical default in Brazilian Portuguese:
- partner → parceiro (not "parceiro(a)", not "parceira" unless context clearly refers to a woman)
- affiliate → afiliado
- referral (person) → indicado
- user → usuário
- member → membro

Do NOT produce constructions like "parceiro(a)", "afiliado/afiliada", or alternating gender across nodes. Pick masculine and keep it consistent throughout.
